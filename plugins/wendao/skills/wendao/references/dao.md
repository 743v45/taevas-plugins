# 道（哲学层）：追问的终极思想

## 核心

穿透表象，抵达本质与结构。

---

## 六大哲学

### 1. 本质主义

抛开表象，追问不变内核。一切可变表象之下，必然存在不变本质。

**核心问题：**
- 它到底是什么？去掉所有装饰后还剩下什么？
- 这个现象背后不变的内核是什么？

### 2. 结构主义

不问"像什么"，问"结构是什么"。结构是可迁移的基础。

**核心问题：**
- 它的结构是怎样的？各部分如何关联？
- 与XX是否结构相同？只是表现形式不同？

### 3. 普遍主义

追求可迁移、可通用、可普适的规律。从特例中寻找通式。

**核心问题：**
- 这个规律在其他领域还成立吗？
- 能否抽象成更通用的模型？

### 4. 溯源主义

不断向上追溯，直到不可再问。找到问题的终极源头。

**核心问题：**
- 再向上追溯一层是什么？
- 它的源头在哪里？

### 5. 层级主义

任何事物都可分层，逐层展开。从表面到核心，层层穿透。

**核心问题：**
- 可以拆成哪几层？哪层是不变的？
- 它的上层抽象是什么？

### 6. 系统主义

单点 → 关系 → 整体 → 闭环。在系统中理解单点。

**核心问题：**
- 它在系统中扮演什么角色？
- 它与其他部分如何相互作用？

---

## 实例：黑匣子的"道"

### 本质主义追问

**表象：** 黑匣子 = 橙色金属盒子 + 抗撞击 + 抗高温 + 水下定位信标

**追问：** 去掉所有这些物理特征，核心是什么？

**答案：** 不变的核心是「数据记录 + 事故重建能力」

### 结构主义追问

**不问：** 它像保险箱吗？

**问：** 它的结构是什么？

**答案：** 传感器 → 存储 → 解码 → 分析

### 普遍主义追问

**问：** 这个模式在其他领域还成立吗？

**答案：**
- 飞机黑匣子 ✓
- 程序日志 ✓
- 审计日志 ✓
- 区块链 ✓（都是数据记录 + 可追溯）

### 溯源主义追问

**问：** 黑匣子的源头是什么？

**答案：** 人类对"事故原因追溯"的需求 → 数据记录需求 → 状态持久化

### 层级主义追问

**问：** 可以拆成哪几层？

**答案：**
- 物理层：抗撞击外壳、防水设计
- 数据层：循环记录、多通道采集
- 应用层：事故分析、原因重建

### 系统主义追问

**问：** 它在系统中扮演什么角色？

**答案：** 航空安全系统的"真相证人"：事故发生 → 黑匣子记录 → 数据提取 → 原因分析 → 系统改进
