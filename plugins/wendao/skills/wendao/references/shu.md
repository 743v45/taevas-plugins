# 术（模式层）：可复用的高级提问范式

## 核心

自动扩展知识、跨领域迁移。

---

## 八大模式

### 1. 本质溯源式

**模式：** 它到底是什么？核心是什么？

**提问：**
- XX的本质是什么？
- 去掉所有装饰后，XX还剩下什么？

### 2. 同构类比式

**模式：** 与XX是否同构？能否类比？

**提问：**
- A和B是否结构相同？
- 它们只是表现形式不同吗？

### 3. 跨域迁移式

**模式：** 放到别的领域还成立吗？

**提问：**
- 这个模式在XX领域是否有对应？
- 能否迁移到YY领域？

### 4. 分层拆解式

**模式：** 可拆为哪几层？哪层不变？

**提问：**
- XX可以拆成哪几层？
- 哪层是不变的，哪层是可变的？

### 5. 归一统摄式

**模式：** 多个现象能否归为一个原理？

**提问：**
- 这些现象能否归一为一个规律？
- 多个XX背后是否有共同原理？

### 6. 边界试探式

**模式：** 它的适用范围与局限是什么？

**提问：**
- 这个方法在什么情况下失效？
- 边界条件是什么？

### 7. 元层次追问

**模式：** 这个方法背后的方法是什么？

**提问：**
- 这个方法的元方法是什么？
- 这背后的元逻辑是什么？

### 8. 置换思想实验

**模式：** 如果换掉场景/主体，会怎样？

**提问：**
- 如果换掉场景，逻辑还成立吗？
- 如果主体换掉，规律还成立吗？

---

## 实例：黑匣子的"术"

### 1. 本质溯源式

**问：** 黑匣子的本质是什么？

**回答：**
- 不是"橙色盒子"
- 不是"抗撞击外壳"
- 本质是「关键状态的持久化与追溯系统」

**追问：** 程序日志的本质是什么？

**回答：** 同样是「关键状态的持久化与追溯系统」

### 2. 同构类比式

**问：** 飞机黑匣子与程序日志是否同构？

**比对：**

| 维度 | 飞机黑匣子 | 程序日志 | 同构性 |
|------|------------|----------|--------|
| 触发 | 传感器采集 | 代码调用 | ✓ |
| 存储 | 循环覆盖 | 文件追加 | ✓ |
| 结构 | 时间序列 | 时间序列 | ✓ |
| 目的 | 事故追溯 | 问题排查 | ✓ |

**结论：** 高度同构，只是实现方式不同

### 3. 跨域迁移式

**问：** 黑匣子模式能否迁移到区块链？

**迁移验证：**
- 飞机黑匣子：记录飞行数据
- 区块链：记录交易数据
- 共同点：不可篡改、可追溯、时间序列

**结论：** 完全可迁移，区块链本质上是"分布式黑匣子"

**问：** 能否迁移到生物学？

**迁移验证：**
- DNA 记录生物体的遗传信息
- 可追溯进化历史
- 可重建"祖先状态"

**结论：** DNA是生物界的"黑匣子"

### 4. 分层拆解式

**问：** 黑匣子可以拆成哪几层？

**分层：**

```
┌─────────────────────────────────────┐
│        应用层（目的）                │
│   事故分析 · 原因重建 · 责任认定     │
├─────────────────────────────────────┤
│        数据层（核心）                │
│   状态记录 · 时间序列 · 不可篡改     │
├─────────────────────────────────────┤
│        传输层（采集）                │
│   传感器 · 采样频率 · 数据编码       │
├─────────────────────────────────────┤
│        物理层（载体）                │
│   抗撞击 · 抗高温 · 防水 · 定位信标  │
└─────────────────────────────────────┘
```

**不变层：** 数据层（状态记录 + 可追溯）
**可变层：** 物理层、传输层、应用层

### 5. 归一统摄式

**问：** 飞机黑匣子、程序日志、审计日志、区块链、DNA，能否归一为一个原理？

**共同原理：**

```
        ┌─────────────────────────┐
        │   状态持久化 + 可追溯    │
        └───────────┬─────────────┘
                    │
        ┌───────────┴─────────────┐
        │                         │
   记录关键状态              时间序列组织
        │                         │
        └───────────┬─────────────┘
                    │
        ┌───────────┴─────────────┐
        │    不可篡改/可验证       │
        └─────────────────────────┘
```

**结论：** 万物皆为"黑匣子"，只是形式不同

### 6. 边界试探式

**问：** 黑匣子模式的边界在哪里？

**边界测试：**

| 场景 | 是否适用 | 原因 |
|------|----------|------|
| 航空事故 | ✓ | 需要追溯 |
| 软件调试 | ✓ | 需要追溯 |
| 金融审计 | ✓ | 需要追溯 |
| 纯随机过程 | ✗ | 无需追溯 |
| 瞬态系统 | ✗ | 状态即逝 |
| 实时控制 | 部分 | 需权衡延迟 |

**结论：** 适用边界是"需要状态追溯的系统"

### 7. 元层次追问

**问：** 黑匣子模式背后的元方法是什么？

**元方法：**

```
┌─────────────────────────────────────┐
│  识别关键状态（什么值得记录）        │
├─────────────────────────────────────┤
│  建立采集机制（如何捕获状态）        │
├─────────────────────────────────────┤
│  确保持久可靠（数据不丢失、不篡改）  │
├─────────────────────────────────────┤
│  设计检索分析（如何用数据重建真相）  │
└─────────────────────────────────────┘
```

**追问：** 这个元方法还能用在哪里？
→ 任何需要"事后重建"的场景

### 8. 置换思想实验

**问：** 如果黑匣子换掉"抗撞击"特性，会怎样？

**推演：**
- 正常飞行：影响不大
- 发生事故：数据损毁
- 结果：失去追溯能力，系统失效

**结论：** "抗撞击"不是本质，但在事故场景下是"必要条件"

**问：** 如果主体从"飞机"换成"人"，会怎样？

**推演：**
- 人的"黑匣子"是什么？
→ DNA（记录遗传信息）
→ 神经突触（记录记忆）
→ 数字足迹（记录行为）

**结论：** 换掉主体，模式依然成立
