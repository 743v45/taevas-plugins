# 行（操作层）：直接可用的提问句式

## 核心

拿来就问，一问就深。

---

## 提问句式库

### 溯源类（追问本质）

| 提问 | 适用场景 |
|------|----------|
| 它的本质结构是什么？ | 理解新概念 |
| 核心是什么？ | 快速抓重点 |
| 去掉所有表象后还剩下什么？ | 去伪存真 |
| 它的源头在哪里？ | 追根溯源 |

### 抽象类（向上提炼）

| 提问 | 适用场景 |
|------|----------|
| 能否抽象成更通用的模型？ | 提炼规律 |
| 这个规律还能再抽象一层吗？ | 深化理解 |
| 从具象到抽象的路径是什么？ | 知识迁移 |

### 同构类（结构比对）

| 提问 | 适用场景 |
|------|----------|
| 与XX是否同构？ | 寻找类比 |
| 它们结构相同吗？只是表现形式不同？ | 验证结构 |
| 能否类比到其他系统？ | 跨域连接 |

### 迁移类（跨域应用）

| 提问 | 适用场景 |
|------|----------|
| 能否迁移到其他领域？ | 知识扩展 |
| 放到生物学/管理学/经济学中还成立吗？ | 验证通用性 |
| 这个模式具有可迁移性吗？ | 判断价值 |

### 归一类（统摄整合）

| 提问 | 适用场景 |
|------|----------|
| 这些能否归一为一个规律？ | 整合知识 |
| 多个现象背后是否有共同原理？ | 发现底层逻辑 |
| 能否用一个框架统摄这些内容？ | 系统化 |

### 分层类（层级拆解）

| 提问 | 适用场景 |
|------|----------|
| 可以拆成哪几层？哪层不变？ | 结构分析 |
| 它的层次结构是怎样的？ | 深入理解 |
| 上层抽象是什么？下层实现是什么？ | 架构设计 |

### 元类（元层次追问）

| 提问 | 适用场景 |
|------|----------|
| 这个方法的元方法是什么？ | 方法论反思 |
| 这背后的元逻辑是什么？ | 深层理解 |
| 能否提炼元结构？ | 抽象提炼 |
| 这是某模式的元版本吗？ | 模式识别 |

### 边界类（极限试探）

| 提问 | 适用场景 |
|------|----------|
| 它的边界在哪里？ | 确定范围 |
| 适用范围与局限是什么？ | 风险评估 |
| 在什么情况下会失效？ | 找反例 |
| 如果换掉场景，逻辑还成立吗？ | 压力测试 |

### 闭环类（抽象-落地）

| 提问 | 适用场景 |
|------|----------|
| 如何从具象向上抽象，再从抽象向下落地？ | 完整闭环 |
| 理论如何指导实践？ | 验证价值 |
| 抽象的结论如何验证？ | 可靠性检验 |

## 实例：黑匣子的"行"

### 初学者提问

**问：** 黑匣子是什么？

**答：** 橙色金属盒子，记录飞行数据。

### "行"层追问（一问就深）

```
用户：我想了解黑匣子

┌─ 溯源类追问 ───────────────────────┐
│ 问：它的本质结构是什么？            │
│ 答：关键状态的持久化与追溯系统      │
└─────────────────────────────────────┘

┌─ 同构类追问 ───────────────────────┐
│ 问：与程序日志是否同构？            │
│ 答：是的，都是状态记录+可追溯       │
└─────────────────────────────────────┘

┌─ 迁移类追问 ───────────────────────┐
│ 问：能否迁移到区块链领域？          │
│ 答：区块链本质是分布式黑匣子         │
└─────────────────────────────────────┘

┌─ 分层类追问 ───────────────────────┐
│ 问：可以拆成哪几层？                │
│ 答：物理层→数据层→应用层            │
└─────────────────────────────────────┘

┌─ 闭环类追问 ───────────────────────┐
│ 问：这个理解能指导什么实践？        │
│ 答：设计任何需要状态追溯的系统      │
└─────────────────────────────────────┘
```

### 快速提问模板

#### 理解新事物（3分钟速成）

```
1. 它的本质是什么？（溯源类）
2. 与XX是否同构？（同构类）
3. 能否迁移到YY领域？（迁移类）
```

#### 深度分析（10分钟深究）

```
1. 它的本质结构是什么？（溯源类）
2. 可以拆成哪几层？哪层不变？（分层类）
3. 这些现象能否归一为一个原理？（归一类）
4. 它的边界在哪里？（边界类）
5. 这个方法的元方法是什么？（元类）
```

#### 跨域创新（寻找机会）

```
1. 它的本质是什么？（溯源类）
2. 能否迁移到其他领域？（迁移类）
3. 换掉主体，规律还成立吗？（边界类+置换实验）
```

## 使用建议

### 灵活组合

不需要全部用完，根据场景选择：

- **快速理解**：溯源类 + 同构类
- **深度分析**：分层类 + 归一类 + 元类
- **创新思考**：迁移类 + 边界类
- **验证理论**：闭环类

### 迭代追问

```
第一问：它的本质是什么？
   ↓
第二问：这个本质的结构是怎样的？
   ↓
第三问：能否迁移到其他领域？
   ↓
第四问：它的边界在哪里？
```

每问一次，理解就深一层。

### 对比验证

用不同类别的提问交叉验证：

```
同构类：与XX是否同构？ → 结构相同 ✓
迁移类：能否迁移到YY？ → 成立 ✓
归一类：能否归一为一个原理？ → 可以 ✓

结论：理解可靠
```
