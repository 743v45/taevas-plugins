# 法（原则层）：追问必须遵守的铁律

## 核心

保证提问不跑偏、不肤浅、不封闭。

---

## 七大原则

### 1. 向上抽象原则

不断拔高一层，不陷在细节。从具体到抽象，从表象到本质。

**检验：** 能否用一句话概括这个事物？

### 2. 向下落地原则

抽象后必须能回到具体。理论必须能指导实践。

**检验：** 抽象的结论能否预测具体现象？

### 3. 跨域迁移原则

能否放到别的领域依然成立。规律的可迁移性是检验其价值的关键。

**检验：** 换个领域，这个规律还成立吗？

### 4. 同构比对原则

外表不同，结构是否一致。抓住结构不变性。

**检验：** A和B外表不同，但结构是否相同？

### 5. 非封闭原则

拒绝"就这样"，保持可扩展。任何结论都是暂时的、可优化的。

**检验：** 这个结论有反例吗？边界在哪里？

### 6. 最小本质原则

删到不能再删，剩下才是核心。奥卡姆剃刀思维。

**检验：** 再删一个元素，系统还成立吗？

### 7. 可证伪/可验证原则

能被检验，不空谈。结论必须具备可验证性。

**检验：** 如何证明这个结论是错的？

---

## 实例：黑匣子的"法"

### 向上抽象原则

**具体：** 橙色盒子、抗压、抗高温、水下信标...

**抽象一层：** 抗灾数据记录设备

**再抽象一层：** 状态持久化系统

**检验：** 能用一句话概括吗？
→ "黑匣子是关键状态的持久化与追溯系统"

### 向下落地原则

**抽象结论：** 状态持久化系统

**落地验证：**
- 飞机黑匣子：飞行数据持久化 ✓
- 程序日志：运行状态持久化 ✓
- 区块链：交易状态持久化 ✓

**检验：** 能否指导实践？
→ 用这个理解，可以设计任何需要"状态追溯"的系统

### 跨域迁移原则

**问：** 黑匣子模式在其他领域还成立吗？

| 领域 | 形式 | 是否成立 |
|------|------|----------|
| 航空 | 物理黑匣子 | ✓ |
| 软件 | 日志系统 | ✓ |
| 金融 | 审计日志 | ✓ |
| 区块链 | 分布式账本 | ✓ |
| 生物学 | DNA（信息记录） | ✓ |

**检验：** 结构相同，只是实现不同

### 同构比对原则

**问：** 飞机黑匣子和区块链，结构相同吗？

| 维度 | 飞机黑匣子 | 区块链 |
|------|------------|--------|
| 本质 | 数据记录 | 数据记录 |
| 特性 | 不可篡改 | 不可篡改 |
| 目的 | 事故追溯 | 交易追溯 |
| 结构 | 传感器→存储→解码 | 交易→区块→链 |

**结论：** 外表不同，结构高度同构

### 非封闭原则

**问：** 黑匣子模式的边界在哪里？

**思考：**
- 只能用于航空吗？ ✗（已扩展到其他领域）
- 只能记录数字数据吗？ ✗（DNA记录生物信息）
- 只能追溯过去吗？ ✗（可用于未来预测）

**结论：** 模式是开放的，边界在探索中扩展

### 最小本质原则

**问：** 黑匣子的最小本质是什么？

**尝试删除：**
- 删除抗撞击外壳？ ✓（软件日志不需要）
- 删除水下信标？ ✓（区块链不需要）
- 删除数据记录？ ✗（核心功能）
- 删除追溯能力？ ✗（核心目的）

**结论：** 最小本质 = 状态记录 + 可追溯性

### 可证伪/可验证原则

**问：** 如何证明"黑匣子模式适用于任何需要追溯的系统"是错的？

**反例测试：**
- 找一个需要追溯但不适合用黑匣子模式的场景？
- 纯随机过程：无需追溯（不适用）
- 瞬态系统：状态即逝，无需记录（不适用）

**结论：** 理论有明确的适用边界，可证伪
