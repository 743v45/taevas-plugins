# 问道体系

## 架构概览

```
┌─────────────────────────────────────────────────────┐
│        道（哲学层）：追问的终极思想
│        目的：穿透表象，抵达本质与结构
├─────────────────────────────────────────────────────┤
│        法（原则层）：追问必须遵守的铁律
│        目的：保证提问不跑偏、不肤浅、不封闭
├─────────────────────────────────────────────────────┤
│        术（模式层）：可复用的高级提问范式
│        目的：自动扩展知识、跨领域迁移
├─────────────────────────────────────────────────────┤
│        行（操作层）：可直接开口用的提问句式
│        目的：拿来就问，一问就深
└─────────────────────────────────────────────────────┘
```

## 交互式使用流程

### 第一步：确认分析对象

**如果用户没有描述要分析什么事物：**

使用 `AskUserQuestion` 工具询问用户：

**结合记忆系统（如果有相关主题）：**
- 检查记忆中是否有适合追问的主题
- 筛选原则：依据新鲜程度和可讨论程度
- 提供记忆主题作为选项

**无记忆时（通用选项）：**
```
问题：您想用什么实例来体验问道体系？或者想分析什么事物？
描述：请描述一个你想深入了解的概念、技术、模式或事物
```

**举例说明：**
- 技术类：缓存、API 接口、状态机、事件驱动
- 概念类：抽象、多态、闭包、异步
- 业务类：用户登录、订单系统、权限管理
- 其他：黑匣子、区块链、DNA、社交网络

**如果用户已经描述了分析对象：**
- 直接进入下一步

**示例：**
```
用户：帮我分析"缓存"
→ 进入第二步
```

→ [详细规则：记忆系统使用](references/jiyi.md)

### 第二步：确定深入层级

**如果用户没有指定层级：**

使用 `AskUserQuestion` 工具询问（支持多选）：

```
问题：您想深入了解哪几层内容？（可多选）
选项：
- 行（操作层）：直接可用的提问句式，拿来就问
- 术（模式层）：可复用的高级提问范式，跨域迁移
- 法（原则层）：追问必须遵守的铁律，不跑偏不肤浅
- 道（哲学层）：追问的终极思想，抵达本质与结构
```

**举例说明（以"缓存"为例）：**
- **行**：我会给你一堆提问句式，你直接用来问"缓存"
- **术**：我会用8种提问模式分析"缓存"，帮你理解它
- **法**：我会用7大原则检验"缓存"这个概念
- **道**：我会从6个哲学角度追问"缓存"的本质

**如果用户指定了具体层级：**
- 按用户指定的层级提供内容

**如果用户只描述了需求但没指定层级：**
- 默认全选中，提供完整四层内容

**示例：**
```
用户：我想深入理解缓存的本质
→ 默认全选，加载全部四层
```

### 第三步：按需加载并分析

根据用户选择，加载对应的 reference 文件并结合用户描述的内容进行分析：

| 用户选择 | 加载文件 | 分析方式 |
|----------|----------|----------|
| 行 | references/xing.md | 用提问句式直接发问 |
| 术 | references/shu.md | 用8种模式分析 |
| 法 | references/fa.md | 用7大原则验证 |
| 道 | references/dao.md | 从6个哲学角度追问 |
| 多选 | 对应多个文件 | 综合分析 |

**分析示例（以"缓存"为例）：**

**行（操作层）分析：**
```
直接提问：
- 缓存的本质结构是什么？
- 缓存能否抽象成更通用的模型？
- 缓存与记忆是否同构？
- 缓存能否迁移到其他领域？
- 缓存可以拆成哪几层？
```

**术（模式层）分析：**
```
本质溯源式：缓存的本质是什么？（空间换时间）
同构类比式：缓存与 CPU 缓存是否同构？
跨域迁移式：能否迁移到人际关系领域？（人际关系缓存）
分层拆解式：可拆为哪几层？（应用层→存储层→介质层）
```

**法（原则层）分析：**
```
向上抽象：缓存 = 空间换时间的通用模式
跨域迁移：CPU 缓存、浏览器缓存、DNS 缓存都成立
同构比对：不同实现，结构相同（快速访问层）
边界试探：什么情况不适合用缓存？（频繁变更的数据）
```

**道（哲学层）分析：**
```
本质主义：去掉技术细节，核心是"预置资源换取响应速度"
结构主义：结构是"慢速存储 + 快速存储 + 替换策略"
普遍主义：在计算机、建筑、人脑中都存在缓存
溯源主义：源头是"时间与空间的权衡"
```

**完整四层输出示例：**
```

│  道：缓存本质是"空间换时间的预置资源系统"          │
├─────────────────────────────────────────────────────┤
│  法：检验缓存在不同场景的可迁移性和边界            │
├─────────────────────────────────────────────────────┤
│  术：缓存与人脑记忆是否同构？能否迁移到人际关系？  │
├─────────────────────────────────────────────────────┤
│  行：它的本质结构是什么？能拆成哪几层？            │
└─────────────────────────────────────────────────────┘
```

## 四层速查

### 道（哲学层）

- 本质主义：追问不变内核
- 结构主义：追问结构而非表象
- 普遍主义：追求可迁移规律
- 溯源主义：追溯终极源头
- 层级主义：逐层穿透
- 系统主义：在系统中理解单点

→ [完整内容：道（哲学层）](references/dao.md)

### 法（原则层）

- 向上抽象原则
- 向下落地原则
- 跨域迁移原则
- 同构比对原则
- 非封闭原则
- 最小本质原则
- 可证伪/可验证原则

→ [完整内容：法（原则层）](references/fa.md)

### 术（模式层）

- 本质溯源式
- 同构类比式
- 跨域迁移式
- 分层拆解式
- 归一统摄式
- 边界试探式
- 元层次追问
- 置换思想实验

→ [完整内容：术（模式层）](references/shu.md)

### 行（操作层）

- 溯源类：它的本质结构是什么？
- 抽象类：能否抽象成更通用的模型？
- 同构类：与XX是否同构？
- 迁移类：能否迁移到其他领域？
- 归一类：这些能否归一为一个规律？
- 分层类：可以拆成哪几层？
- 元类：这个方法的元方法是什么？
- 边界类：它的边界在哪里？
- 闭环类：如何从具象向上抽象，再向下落地？

## 实践原则

1. **永不满足表象**：每追问一次，都离本质更近一步
2. **结构优先**：先搞懂结构，再看细节
3. **不断验证**：每个结论都寻找反例验证
4. **保持开放**：承认理解永远可深化，不封闭
5. **注重迁移**：检验规律的可迁移性，判断价值
