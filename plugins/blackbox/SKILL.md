# 黑匣子 (BlackBox)

## 核心定义

```
黑匣子 = 飞行记录仪
```

一个运行时观测系统，默默记录代码运行时的一切——性能数据、资源使用、网络请求、错误事件。出事后可完整回放分析，让问题从"难以复现"变成"有迹可循"。

---

## 与传统监控的区别

| 维度 | 传统监控 | 黑匣子 |
|-----|---------|--------|
| **查看时机** | 实时持续查看 | 出事后才查看 |
| **信息呈现** | 主动推送输出 | 被动按需查询 |
| **正常状态** | 产生大量噪音 | 完全静默 |
| **异常状态** | 告警淹没在噪音中 | 完整上下文清晰可见 |

**核心理念：** 平时沉默，出事说话

## 快速开始

### 1. 了解哲学

**[philosophy.md](philosophy.md)** - 理解黑匣子的六大核心理念

```
事后可见性 → 让不可见的过程在事后可见
客观中立性 → 只记录事实，不干预运行
生存优先性 → 记录系统可靠性高于被记录系统
证据唯一性 → 记录不可篡改，成为唯一可信依据
全局追溯性 → 所有行为可溯源，责任可定
迭代预防性 → 用历史记录反推问题根源
```

### 2. 遵循原则

**[principles.md](principles.md)** - 遵循通用记录原则

```
完整性 → 关键过程不遗漏
不可篡改 → 记录一旦写入不可修改
可靠性 → 极端环境下仍能保存
实时性 → 数据同步记录，不延迟
可溯源性 → 每条记录带完整上下文
最小必要 → 只记录关键参数
可读性 → 标准格式，可被解析
防丢失 → 多副本、循环覆盖
独立性 → 不依赖主系统
合规审计 → 满足监管要求
```

### 3. 选择模式

**[runtime.md](runtime.md)** - 选择适合的运行模式

| 场景 | 推荐模式 |
|-----|---------|
| 生产环境默认 | OFF（关闭） |
| 生产故障追踪 | MINIMAL（最小） |
| 测试环境 | STANDARD（标准） |
| 生产事故分析 | FULL（完整） |
| 开发调试 | DEBUG（调式） |

### 4. 开始记录

**[operations.md](operations.md)** - 使用黑匣子操作方法

```typescript
// blackboxEnabled 由 Feature Flag 模式提供
if (blackboxEnabled) {
  // 📦 [黑匣子] 性能 - 记录操作耗时
  blackbox.record('performance', {
    operation: 'login',
    duration: 234,
    timestamp: Date.now(),
  });
}
```

## 埋点设计决策树

选择要观测的内容时，按以下路径判断：

```
需要观测什么？
    │
    ├── 性能问题？ → 见 [性能监控模式](patterns/performance.md)
    ├── API 问题？ → 见 [网络拦截模式](patterns/network.md)
    ├── 偶发失败？ → 见 [自动重试模式](patterns/retry.md)
    ├── 状态验证？ → 见 [断言工具]

    ├── 错误诊断？ → 见 [上下文收集]
    └── 团队协作？ → 见 [可观测性]
```

## 环境开关控制

黑匣子的启用由 **Feature Flag 模式**控制，详见：

**[../feature-flag/SKILL.md](../feature-flag/SKILL.md)**

```bash
# 使用 Feature Flag 控制黑匣子
BLACKBOX_ENABLED=true pnpm test

# 配置运行模式
BLACKBOX_ENABLED=true BLACKBOX_LEVEL=full pnpm run
```

## 埋点注释规范

### 标记格式

```typescript
// �[黑匣子] <类型> <说明>
```

### 类型分类

| 标记 | 类型 | 说明 |
|-----|------|------|
| `📦 [黑匣子] 性能` | 性能埋点 | 记录操作耗时 |
| `📦 [黑匣子] 网络` | 网络埋点 | 记录请求/响应 |
| `📦 [黑匣子] 资源` | 资源埋点 | 记录内存/CPU |
| `📦 [黑匣子] 错误` | 错误埋点 | 记录异常信息 |
| `📦 [黑匣子] 状态` | 状态埋点 | 记录页面状态 |
| `📦 [黑匣子] 快照` | 快照埋点 | 保存现场数据 |

### 搜索命令

```bash
# 搜索所有黑匣子埋点
grep -r "📦 \[黑匣子\]" src/

# 搜索特定类型
grep -r "📦 \[黑匣子\] 性能" src/
grep -r "📦 \[黑匣子\] 网络" src/

# 排除黑匣子查看正常代码
grep -v "📦 \[黑匣子\]" src/file.ts
```
